## 3. 接口封装方案

为了方便后续接入app、web页面或OpenClaw等工具，建议封装以下接口：

### 3.1 RESTful API 接口
- **基础认证**：JWT Token
- **数据格式**：JSON
- **核心接口**：
  - `/api/v1/strategies` - 策略管理（查看、添加、修改、删除）
  - `/api/v1/orders` - 订单管理（创建、查询、取消）
  - `/api/v1/positions` - 持仓查询（当前持仓、历史持仓）
  - `/api/v1/ai/analysis` - AI分析（获取分析结果、触发分析）
  - `/api/v1/system/status` - 系统状态（服务状态、组件健康）
  - `/api/v1/notifications` - 通知管理（获取通知、设置通知偏好）
  - `/api/v1/data` - 数据查询（K线数据、历史数据）

### 3.2 WebSocket 接口
- 实时行情数据推送
- 交易信号实时通知
- 系统状态变化通知

### 3.3 消息队列接口
- 支持与OpenClaw等外部系统通过消息队列通信
- 事件驱动的架构设计

### 3.4 接口实现步骤
1. **创建API模块**：在src目录下创建api模块
2. **实现认证系统**：JWT token生成与验证
3. **开发核心接口**：实现上述RESTful API
4. **添加文档**：使用Swagger或类似工具生成API文档
5. **测试接口**：确保所有接口正常工作
6. **部署API服务**：可与主系统一起部署或单独部署
